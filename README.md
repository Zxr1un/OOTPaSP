# Небольшая инструкция
Сейчас в репозитории 4 VS проекта:
- Рефакторинг версии с 1 по 4 лабораторной
- Проект 5 лабораторной 
- Проект библиотеки со всеми интерфейсами 5 лабораторной работы (нужна, чтобы основной проект компилировался, не зная про библиотеку)
- Проект библиотеки для 5 лабораторной (загружается динамически в MainWindow в основном проекте 5 лабы)

## Осторожно
- В проекте очень сильная зависимость от WPF, поэтому для работы DLL используются интерфейсы для окон и заготовка для фабрики окон.
- А так же в проекте сцена является потомком основного класса FigureMy. Поэтому механизм сохранения у неё такой же, как и у остальных фигур.
- После переход на динамическую загрузку библиотеки, появилась вспомогательная библиотека интерфейсов (подключается обычной using, но можно отключить и все интерфейсы поместить в основной проект)

## Основные особенности управления
- Редкатор: ПКМ по фигуре
- Выделение: shift (работает в т.ч. в дереве иерархии)
- Выделение в объединении: ctrl (выделит границы текущей фигуры в объединении и границы всех родителей)
- Двойной клик по объекту в дереве иерархии: открыть его редактор

## В редакторе
- будьте внимательны, есть кнопка "сохранить" (сохранить шаблон фигуры для текущей сесии) и "сохранить в файл" (сохраняет фигуру в файл json)
- Если вам нужен родитель фигуры, сделать это можно либо через кнопку "перейти к родителю" или через дерево иерархии
- Чтобы отвязаться от родителя, нажмите соответствующую кнопку в редакторе фигуры
- Чтобы удалить фигуру, нажмите соответствующую кнопку в редакторе фигуры

## В верхнем меню
- В "пункте файл": очистка сцены, сохранение сцены, загрузка сцены/фигуры
- В пункте "создать фигуру": 6 основных типов фигур (многоугольник -- это полилиния)
- В пункте "сохранённые" лежат копии фигур, которые вы сохранили в текущей сесии или загрузили из файла в текущей сесии
- Пункт "объеденить" появляется после выбора нескольких фигур доступных для объединнения
